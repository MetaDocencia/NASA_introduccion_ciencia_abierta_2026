## Descripción general

En esta lección se revisa la definición de Ciencia Abierta y de otros términos frecuentes como productos de investigación, datos, código y resultados. Conocer estas definiciones prepara para participar y contribuir al mundo de la Ciencia Abierta. Además, se presentan ejemplos que muestran cómo se usan en la práctica algunas de las herramientas de Ciencia Abierta. La lección cierra con un estudio de caso en el que un grupo compartió de manera abierta sus datos, resultados, código y artículo, y cómo esto benefició de manera importante su trabajo y sus actividades de difusión.

## Objetivos de aprendizaje

Al finalizar esta lección se debería poder:

- Definir tipos frecuentes de productos de investigación, incluyendo datos, código y resultados.
- Enumerar formas habituales de compartir datos, código y resultados al practicar Ciencia Abierta.

## Definición de Ciencia Abierta y productos de investigación

### Productos abiertos de investigación

El conocimiento científico, o productos de investigación, adopta la forma de datos, resultados, código y software.

Dentro de estos productos de investigación se incluyen además otros tipos de productos, como metodologías, algoritmos y artefactos físicos.

### ¿Qué son los datos?

En términos generales, los datos son piezas de información sobre un objeto de estudio. Esto incluye verdades teóricas, mediciones sin procesar o valores altamente procesados.

También puede haber datos acerca de los datos, conocidos como metadatos. En estas lecciones, cuando se habla de datos se hace referencia a información científica o técnicamente relevante que puede almacenarse en formato digital y a la que se puede acceder de manera electrónica, como por ejemplo:

- Información producida por misiones y experimentos, incluyendo calibraciones, coeficientes y documentación.
- Información necesaria para validar las conclusiones científicas de publicaciones revisadas por pares.

Los datos abiertos pueden tener muchas características. Pueden incluir metadatos ricos y robustos y estar disponibles en una variedad de formatos. Estas características se detallan más adelante en este módulo y se desarrollan aún más en el Módulo 3, «Datos Abiertos».

### ¿Qué es el código?

Muchas personas dedicadas a la ciencia escriben código fuente para producir software que analiza datos o modela observaciones. El código es un lenguaje que seres humanos pueden escribir y entender. El software suele ser una colección de programas, datos y otra información que un sistema informático utiliza para realizar tareas específicas. En su trabajo, las personas que hacen ciencia escriben y usan muchos tipos de software distintos.

**Software de propósito general**

Software producido para un uso amplio, no específicamente para fines científicos. Esto incluye tanto software comercial como software de código abierto.

**Software operativo y de infraestructura**

Software utilizado por centros de datos y grandes instalaciones de tecnologías de la información para ofrecer servicios de datos.

**Bibliotecas**

Conjunto de código reutilizable ya escrito, que contiene implementaciones estandarizadas de algoritmos frecuentes y utilidades para tareas como análisis estadístico, manipulación de datos y visualización, entre otras. Estas bibliotecas están pensadas para integrarse en sistemas de software más grandes y permiten que quienes desarrollan software incorporen funcionalidades complejas sin necesidad de programarlas desde cero.

**Software de modelado y simulación**

Software que implementa soluciones a ecuaciones matemáticas a partir de datos de entrada y condiciones de contorno, o que infiere modelos a partir de datos.

**Software de análisis**

Software desarrollado para manipular mediciones o resultados de modelos con el fin de visualizarlos o comprenderlos mejor.

**Software de uso único**

Software escrito para una situación particular, por ejemplo para hacer una figura para un artículo o para manipular datos de una forma muy específica.

Algunas de las herramientas que se pueden usar para desarrollar software se presentan en la Lección 4. Entender cómo encontrar y usar código de otras personas, cómo crear código propio y cómo compartirlo son partes importantes del avance científico, tema que se desarrollará en el Módulo 4, «Código Abierto».

### ¿Qué son los resultados?

Los resultados reúnen los distintos productos de investigación del proceso científico. Las publicaciones son el tipo de resultado más conocido, pero también se incluyen muchos otros tipos de productos. Tanto los datos como el software pueden considerarse resultados, pero cuando hablemos de resultados en este contexto pondremos el foco en otros tipos de productos. Los resultados pueden incluir lo siguiente:

- Publicaciones revisadas por pares.
- Cuadernos computacionales.
- Entradas de blog.
- Videos y pódcast.
- Publicaciones en redes sociales.
- Resúmenes y presentaciones en congresos.
- Discusiones en foros.

Es posible que el ciclo de vida de la investigación ya sea familiar, pero no siempre se conocen los distintos tipos de resultados que pueden compartirse de manera abierta en cada etapa del proceso. Al compartir resultados, se busca ser lo más abierto posible, con el objetivo de aumentar la reproducibilidad, la disponibilidad y la transparencia de la ciencia. A lo largo del ciclo de vida de la investigación, habrá múltiples oportunidades de compartir diferentes resultados de forma abierta, lo que puede dar lugar a nuevas colaboraciones y nuevas líneas de trabajo. Se ofrecen más detalles sobre el alcance de los resultados abiertos en el Módulo 5, «Resultados Abiertos».

## Uso de herramientas de Ciencia Abierta en la práctica

Las siguientes lecciones de este módulo exploran diferentes herramientas y recursos disponibles para quienes investigan y desean usar, producir y compartir Ciencia Abierta. Como se mencionó, es importante pensar cómo integrar los principios de Ciencia Abierta en todas las etapas del proceso de investigación. A continuación se presenta una descripción general de una manera posible de articular las distintas piezas.

### Componentes de la Ciencia Abierta

[IMG: https://drive.google.com/file/d/1IduduABTYGsC8rV2T752x_TC41JaPpbU/view?usp=drive_link]

Los cuatro componentes principales de la Ciencia Abierta se pueden organizar en una pirámide de productos de investigación compartidos abiertamente.

El artículo de investigación, estrechamente vinculado con los resultados, se ubica en la parte superior de la pirámide y resume la forma en que se combinaron el software y los datos para producir esos resultados.

La práctica de compartir estos componentes puede darse con diferentes niveles de completitud. Para ofrecer una guía acerca de cómo compartir los componentes de la Ciencia Abierta, se simplifica este rango de completitud en tres niveles: «bueno», «mejor» y «óptimo». Este rango refleja el grado de compromiso con la difusión abierta de la Ciencia Abierta en todas las etapas del proceso de investigación y para todos sus productos.

### Compartir datos abiertos

Los datos pueden compartirse fácilmente mediante muchos servicios distintos. La forma más adecuada de compartir datos científicos suele ser a través de un repositorio de datos de largo plazo que permita tanto preservar los datos como hacerlos localizables. La imagen siguiente muestra algunas consideraciones para compartir datos a través de [Zenodo](https://zenodo.org/) (enlace externo), un repositorio generalista de datos. Estas consideraciones son similares a las de otros repositorios de datos. En el Módulo 3, «Datos Abiertos», se ofrecen más detalles sobre cómo compartir datos abiertos.

### Compartir código abierto

Cuando se comparte código, a menudo se hace mediante una plataforma en línea con control de versiones, que permite que otras personas contribuyan al código y que además conserva un historial de los cambios realizados. Por ejemplo, muchas personas que investigan optan por publicar sus archivos de código en [GitHub](https://github.com/) (enlace externo) con una licencia BSD de 3 cláusulas. Esto permite que otras personas contribuyan y reutilicen el código. Los pasos para preservar el código y hacerlo localizable se presentan en el Módulo 4, «Código Abierto».

<img src="../images/media/image9.png" style="width:100%;height:auto;" />

### Compartir un artículo abierto

Quienes investigan pueden optar por publicar en una revista que utilice una licencia de acceso abierto. Es posible buscar revistas de acceso abierto mediante el Directory of Open Access Journals (DOAJ). Este tema se aborda con más detalle en el Módulo 5, «Resultados Abiertos».

### Compartir resultados abiertos

Al compartir resultados, es importante incluir la metodología utilizada para producirlos, es decir, la «proveniencia», directamente junto con el código. El código tiende a evolucionar con el tiempo, mientras que los resultados que produce pueden mantener cierta estabilidad. Por este motivo, compartir la metodología ayuda a que otras personas puedan reproducir resultados que envejecen con código más reciente, incluso si los pasos específicos para producirlos cambian a medida que el código evoluciona.
