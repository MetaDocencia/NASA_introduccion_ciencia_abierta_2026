# Introducción a la Ciencia Abierta Módulo 3: Datos Abiertos

Te damos la bienvenida a Introducción a la Ciencia Abierta: Datos Abiertos

## Acerca de este módulo

Este módulo se enfoca en la práctica y aplicación de la ciencia abierta para los datos. Proporciona un proceso de "cómo hacer" para encontrar y evaluar datos abiertos para su uso, para producir datos abiertos y para compartir datos abiertos. Los flujos paso a paso son fáciles de seguir y pueden utilizarse como listas de verificación una vez que completes el módulo. Algunos de los temas clave que se discuten incluyen: planes de gestión de datos, el proceso para evaluar datos para su reutilización, la creación de un plan para producir datos que incluya la elección de formatos abiertos y el agregado de documentación, las consideraciones para compartir datos y cómo hacer que tus datos sean citables.

### Objetivos de aprendizaje

Luego de completar este módulo, deberías poder:

- Describir el significado y el propósito de los datos abiertos, sus beneficios y cómo se utilizan los principios FAIR.
- Recordar métodos para evaluar la reutilización de los datos a partir de su documentación y citar los datos según se indique.
- Implementar un plan de gestión de datos abiertos; seleccionar formatos de datos abiertos; agregar la documentación necesaria, incluyendo metadatos, archivos README y control de versiones; y hacer que los datos sean reutilizables y encontrables.
- Evaluar si tus datos deberían y pueden compartirse.
- Recordar prácticas para hacer que los datos estén más disponibles, incluyendo el registro de un DOI afiliado y las instrucciones de citación en la documentación.

## Términos clave

Estos términos clave son temas importantes para este módulo. Selecciona el término para ver la descripción.

**Copyright** – Un tipo de propiedad intelectual que protege obras originales de autoría tan pronto como un autor fija la obra en una forma tangible de expresión. Muchos tipos distintos de obras están cubiertos por la ley de copyright, incluyendo productos de datos y software (así como libros, poemas, pinturas, fotografías, ilustraciones, composiciones musicales y muchos más).

Nota: los datos crudos, que se consideran hechos, no están cubiertos por la ley de copyright.

**Datos** – Información fáctica (como mediciones o estadísticas) utilizada como base para el razonamiento, la discusión o el cálculo.

**Licencia de datos** – Las licencias de datos brindan a cualquier creador de datos una forma de otorgar al público permiso para usar sus productos bajo la ley de copyright. Del mismo modo, las licencias de datos dan a quienes usan datos pautas claras sobre cómo pueden reutilizar el material.

**Productos de datos** – Los productos de datos son activos reutilizables que procesan datos y generan hallazgos que ayudan a las organizaciones a tomar mejores decisiones. Los productos de datos pueden incluir conjuntos de datos, flujos de datos, fuentes de datos, APIs, código, modelos de datos, modelos analíticos y tableros.

**Licencias CC-BY y CC0** – CC-BY y CC0 son licencias de datos de Creative Commons. CC-BY permite a quienes reutilizan distribuir, remezclar, adaptar y construir sobre el material en cualquier medio o formato, siempre que se dé atribución al creador. Esta licencia permite el uso comercial. CC0 permite a los creadores renunciar a su copyright y colocar sus obras en el dominio público mundial. CC0 permite a quienes reutilizan distribuir, remezclar, adaptar y construir sobre el material en cualquier medio o formato, sin condiciones.

**Plan de gestión de datos** – Un plan de gestión de datos (DMP, por sus siglas en inglés) describe el qué, dónde, cuándo y quién de los datos que se crearán durante un proyecto de investigación. Componentes comunes de los planes de gestión de datos incluyen una descripción del tipo, volumen y formato de los datos; dónde y cuándo los datos estarán disponibles; y quién hará que los datos estén disponibles. El plan también puede describir variables de datos, fuentes, exactitud y precisión si esa información está disponible.

**Metadatos** – Datos que describen datos. Pueden ser globales, describiendo el contenido general de un solo archivo o de una colección de archivos, o locales, describiendo una variable individual dentro del archivo. En general, los metadatos globales ofrecen información sobre quién creó el archivo, información sobre el conjunto de datos, qué satélite(instrumento)(laboratorio)(etc.) creó el conjunto, el DOI y la información del formato del archivo, entre otros campos de metadatos. Los metadatos locales sobre variables contienen información como el nombre completo(largo) de la variable, factores de escala o información de incertidumbre, y unidades de medida.

**Identificadores persistentes (PID)** – Una cadena única que identifica un objeto, como un conjunto de datos. Aunque la ubicación en línea del objeto pueda cambiar, el PID no cambiará y también conducirá de regreso a los datos, garantizando que las citas que referencian el PID siempre sean válidas.

**Encontrables (datos)** – Datos que pueden descubrirse fácilmente tanto por personas como por máquinas. Deberían incluir un identificador persistente único, metadatos ricos que describan los datos y el contexto, y estar registrados en un índice que sea buscable.

**Accesibles (datos)** – Datos a los que se puede acceder a través de protocolos estándar de comunicación, con metadatos a los que se puede acceder incluso si los datos en sí ya no están disponibles.

**Interoperables** (datos) – Datos que usan ontologías y vocabularios controlados para que puedan usarse y(o) combinarse con otros conjuntos de datos relevantes en distintas aplicaciones.

**Reutilizables (datos)** – Datos que tienen una licencia clara, procedencia detallada, descripción(definición) adecuada, cumplen con estándares de la comunidad(del dominio) y pueden replicarse o combinarse con otros datos.
