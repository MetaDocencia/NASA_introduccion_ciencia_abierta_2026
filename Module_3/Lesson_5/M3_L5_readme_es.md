## Descripción general

En esta lección, practicarás la escritura de un plan de gestión de datos (DMP) para tu proyecto. Contar con un DMP puede aclarar dónde, cuándo y cómo tus datos pueden ser reutilizados. También aprenderás cómo involucrarte en comunidades de datos abiertos, abriendo nuevas vías para compartir tus datos. Por último, conocerás recursos que puedes explorar, así como capacitaciones que puedes realizar para comenzar tu camino con datos abiertos.

## Objetivos de aprendizaje

Luego de completar esta lección, deberías poder:

- Describir los pasos a seguir al escribir un plan de gestión de datos.
- Enumerar oportunidades para involucrarte en comunidades de datos abiertos.
- Identificar recursos adicionales de ciencia abierta y enumerar formas de continuar tu formación.

## Escritura de un Plan de Ciencia Abierta y Gestión de Datos

A lo largo de este módulo se presentaron el proceso, las responsabilidades y los factores a considerar al crear un Plan de Ciencia Abierta y Gestión de Datos. Elementos comunes de los DMP relevantes para datos abiertos incluyen una descripción de lo siguiente:

|  |  |
|---|---|
| ¿Qué? | Formatos de datos y (cuando corresponda) estándares |
| ¿Cuándo? | Cuándo y si compartir datos |
| ¿Dónde? | Repositorios previstos para datos archivados |
| ¿Cómo? | Cómo el plan habilita la reutilización de los datos |
| ¿Quién? | Roles y responsabilidades de los miembros del equipo para implementar el DMP |

Dos muy buenos lugares para comenzar son [https://dmptool.org/](https://dmptool.org/) (enlace externo) y [https://dmponline.dcc.ac.uk/](https://dmponline.dcc.ac.uk/) (enlace externo). Necesitarás crear un inicio de sesión gratuito para usar estas herramientas, pero ambos sitios guían a quienes investigan a través de los pasos para escribir un DMP. Incluso existen plantillas de DMP ya disponibles en DMP Tool, como la [plantilla de DMP](https://science.nasa.gov/researchers/templates-planetary-science-division-appendix-c-roses-proposals/) de la División de Ciencias Planetarias de la NASA. También hay ejemplos públicos de planes de gestión de datos en [https://dmptool.org/public_plans](https://dmptool.org/public_plans) (enlace externo) y [https://dmponline.dcc.ac.uk/public_plans](https://dmponline.dcc.ac.uk/public_plans) (enlace externo).

Si estás solicitando financiamiento, es muy probable que existan requisitos específicos detallados en la convocatoria. Por ejemplo, el financiador puede exigir una licencia determinada o el uso de un repositorio específico. Asegúrate de contrastar tu plan con esos requisitos.

### Actividad 5.1: Revisar un plan de gestión de datos

Revisa el ejemplo de un plan de gestión de datos público del proyecto [ThEBES](https://dmptool.org/plans/30771/export.pdf?export%5Bquestion_headings%5D=true) (enlace externo).

En el DMP, responde las siguientes preguntas:

1. Qué: Formatos de datos y (cuando corresponda) estándares.
2. Cuándo: Cuándo y si compartir datos.
3. Dónde: Repositorios previstos para datos archivados.
4. Cómo: Cómo el plan habilita la reutilización de los datos.
5. Quién: Roles y responsabilidades de los miembros del equipo para implementar el DMP.

## Resumen de Datos Abiertos

Felicitaciones. Ahora que completaste el módulo, deberías poder hacer lo siguiente:

- Explicar qué significa datos abiertos, sus beneficios y cómo se usan los principios FAIR.
- Descubrir datos abiertos, evaluar los datos para su reutilización revisando la documentación provista y citar los datos según las instrucciones.
- Crear un plan de gestión de datos abiertos, seleccionar formatos de datos abiertos y agregar la documentación necesaria como metadatos, archivos README y control de versiones para que los datos sean reutilizables y encontrables.
- Evaluar si los datos pueden y deben compartirse, y usar el proceso de accesibilidad de datos (por ejemplo, agregar un DOI e instrucciones de citación para que sean encontrables y citables).
